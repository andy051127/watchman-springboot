지금부터 study-session을 제작할거야. 아래 내용을 참고해서 만들어보자.
절대로 한번에 진행하지 말고 진행 순서대로 나누어 진행해보자
나한테 추가적으로 필요한 정보가 있다면 꼭 물어봐줘

- 구현 순서:
	1. DB --> Domain / Service / Repository / Controller (따로따로) --> html

- 페이지 구성:
1. study-group.html: 
	- 참여중인 그룹 목록 나열
	- 그룹 추가/참가
2. study-group-info.html:
	- 그룹 상세 정보들, 참가자 목록 및 집중도 랭킹 표시 
	- "스터디룸 들어가기" 버튼을 통해 스터디룸 세션 진입
	- (방장) 강퇴, 방 종료
3. study-group-session:
	- 화상회의 프로그램 "zoom" 처럼 다른 사람의 화면이 보이고, 참가 인원수에 따라 배치가 유동적으로 달라짐
	- 우측에 참가자들과 실시간 소통을 할 수 있는 영역이 있어야함.
	- 채팅은 누가 보냈는지 프로필사진과 이름이 뜨도록 만들어야함.
	- 카메라를 감싸는 UI를 통해 집중/딴짓 상태 실시간 전송

- 기능:

1. 그룹 생성 / 참가 / (방장)퇴장 / (방장)삭제
2. 실시간 집중 상태 공유(WebSocket)
3. 서로의 카메라가 보임(WebSocket)
4. 실시간 채팅 기능(WebSocket)
