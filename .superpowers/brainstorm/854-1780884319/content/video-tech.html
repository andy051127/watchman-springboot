영상 공유 방식 선택

스터디룸에서 서로의 카메라를 어떻게 공유할까요?

A

WebRTC (권장)

브라우저 ↔ 브라우저 직접 P2P 연결 — 실제 Zoom / Google Meet 방식

장점

  • 화질 좋고 지연 낮음
  • 기존 MediaPipe 얼굴 감지와 그대로 호환
  • 서버 부하 거의 없음
  • Spring WebSocket은 연결 중개(시그널링)만 담당

단점

  • 구현 복잡도 높음
  • 외부 STUN 서버 필요 (Google 무료 서버 사용 가능)
B

WebSocket 프레임 전송

카메라 캔버스 캡처 → 서버 → 전체 broadcast

장점

  • 구현 단순
  • Spring WebSocket 하나로 통일

단점

  • 화질 낮고 지연 높음
  • 서버 트래픽·부하 매우 높음
  • 참가자 많을수록 급격히 느려짐
