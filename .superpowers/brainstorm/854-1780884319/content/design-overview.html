🔌 WebSocket 채널 (STOMP)

Topic 용도 Payload
/topic/room/{id}/presence 입장 / 퇴장 알림 userId, nickname, avatar, action
/topic/room/{id}/focus 집중/딴짓 상태 브로드캐스트 userId, focused (boolean)
/topic/room/{id}/chat 채팅 메시지 userId, nickname, avatar, message, timestamp
/topic/room/{id}/signal WebRTC 시그널링 (offer/answer/ICE) from, to, type, data

Method Endpoint 설명

💾 세션 저장

그룹 스터디룸을 나갈 때 개인 집중 데이터(focusedTime, distractedTime)를 기존 sessions 테이블에 저장합니다. 솔로 세션과 동일한 방식.

📷 WebRTC 시그널링 흐름

1. A가 룸 입장 → presence 브로드캐스트
2. 기존 참가자 B가 A에게 SDP offer 전송 (signal 채널)
3. A가 B에게 SDP answer 응답
4. 양측 ICE candidate 교환
5. P2P 영상 스트림 연결 완료 — 이후 서버 경유 없음
